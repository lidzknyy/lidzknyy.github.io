面向对象的三大特性

Java 面向对象编程的三大特性：封装、继承、多态。

封装：通过访问控制修饰符实现。隐藏类的属性和实现细节，仅对外提供公共访问方法和变量，提高复用性和安全性。

继承：通过 extends 实现。使用已存在的类的定义为基础扩展新的类，提高代码的复用性。

多态：父类引用指向子类对象。实现多态的三个必要条件：继承、重写、向上转型。

接口 和 抽象类 的异同

抽象类是用来捕捉子类的通用特性。接口是抽象方法的集合。

从设计层面来说，抽象类是对类的抽象，是一种模板设计。接口是行为的抽象，是一种行为的规范。

相同点

  • 接口和抽象类都不能实例化
  • 抽位于继承的顶端，用于被其他实现或继承
  • 都包含抽象方法，其子类都必须覆写这些抽象方法

不同点

抽象类 接口
声明 abstract class interface
实现 子类使用 extends 继承抽象类。如果子类不是抽象类，则必须提供抽象类中所有抽象方法的实现。 实现类使用implements 实现接口。实现类需要提供接口中所有抽象方法的实现。
构造器 可以有构造器 不能有构造器
static 代码块 可以有 static 代码块 不能有 static 代码块
访问修饰符 所有方法可以使用任意访问修饰符 接口方法默认 public abstract。且不允许 private 和 protected 修饰。
多继承 一个类最多只能继承一个抽象类 一个类可以实现多个接口
字段声明 抽象类的字段声明可以是任意的 接口中的字段默认都是 public static final

注：Java 8 中接口引入默认方法和静态方法，以此来减少抽象类和接口之间的差异。默认方法使用 default 修饰，可以有方法体。

关于抽象类和接口使用选择：

  • 行为模型应该总是通过接口而不是抽象类定义，所以通常是优先选用接口，尽量少用抽象类。
  • 选择抽象类的情况：需要定义子类的行为，又要为子类提供通用的功能。

多态体现在哪些方面？

重载和重写。

  • 重载体现编译时的多态性，在编译时就能确定调用的方法。
  • 重写体现运行时的多态性，在编译时不能确定，运行时才能确定调用的方法。

重载 和 重写 的区别

  • Override 是子类实现接口，或者继承父类时，保持 方法签名（方法名称+参数列表）完全相同，返回值类型 <= 父类，异常 <= 父类，访问修饰符 >= 父类， 实现不同的方法体，是 垂直方向上行为 的不同实现。重写在编译期无法确定方法调用。

  • Overload 是 方法名称相同，方法参数类型或个数不同，是 水平方向上行为 的不同实现。重载在编译期就可以确定方法调用。

重载的方法能否根据返回类型区分？

不能。重载体现编译时的多态性，如果仅有返回类型不同，则不能确定 接收该方法返回值的变量 的类型，存在二义性，不能通过编译。

IOC、AOP

IOC：控制反转。改变传统开发中主动创建对象的方式，将对象交给第三方容器创建和管理。起到降低耦合的作用。

AOP：面向切面编程。对面向对象编程的一种增强。将那些与业务无关，却为业务模块多共同调用的逻辑或责任封装起来，减少系统的重复代码，降低模块之间耦合度，提高维护性。
使用场景：事务管理、安全检查、权限控制、数据校验、日志、缓存、对象池管理等。

死锁的产生、条件、解除

死锁 (deadlock) ：指两个或两个以上的进程/线程在执行过程中，因争夺资源而造成的一种互相等待的现象，若无外力作用，他们都将无法推进下去。例如：线程A占有了资源1并请求资源2，线程B占有了资源2并请求资源1，且A、B都不愿意释放资源，此时就发生了死锁。

产生死锁的原因

  • 进程推进顺序不当
  • PV操作使用不当
  • 同类资源分配不均或对某些资源的使用未加限制……等

产生死锁的四个必要条件

（1）互斥条件：一个资源每次只能被一个进程使用。

（2）占有和等待条件：进程在请求资源而得不到满足时，不释放已占有资源。

（3）不剥夺条件：进程已获得的资源，在未使用完之前，不能强行剥夺。

（4）循环等待条件：若干进程之间形成一种头尾相接的循环等待资源关系。

前三个条件是死锁存在的必要条件，但不是充分条件。第四个条件是前三个条件同时存在时产生的结果。只要能破坏4个必要条件之一，就可以防止死锁。

死锁的解除和预防

破坏必要条件

只要能破坏四个必要条件之一，就能防止死锁。

  • 破坏互斥条件

  使资源可同时访问而非互斥使用。但如可写文件、键盘等设备的特殊性质决定其只能互斥的占有，而不能被同时访问，所以该方法很多时候都行不通。

  • 破坏占有和等待条件

  静态分配策略。进程在执行前就申请需要的全部资源，直至所要的资源都得到满足后才开始执行。该方法会严重降低资源利用率。

  例如：进程 A 占有资源R1、R2，进程 B 需要申请资源 R2，但资源 R2 直至 A 快执行结束时才会使用，在 A 执行完之前，B 就需要一直等待。

  • 破坏不剥夺条件

  该方法只适用于内存和处理器资源。

  方法一：进程若要申请资源，必须先释放掉已占用资源，如果仍需使用该资源，应向系统重新提出申请。该方法会造成重复的申请和释放资源。

  方法二：进程申请新资源时，如果系统有则分配，否则剥夺进程已占有的资源，让进程进入等待状态，资源充足时再重新唤醒。

  • 破坏循环等待条件

    层次分配策略。将系统所有资源排列到不同层次中，进程申请资源时必须满足如下规则：

    • 得到某层资源后，只能申请较高一层的资源；

    • 释放某曾资源时，必须先释放多占用的所有更高层资源；

    • 得到某曾资源后，如果想申请同层其它资源，必须先释放掉此层已占有资源。

  按序分配策略。对系统所有资源按顺序编号。进程申请资源时，必须以上升的次序进行申请。例如：系统共有 n 个进程，m 个资源，r~i~ 表示第 i 个资源，于是 m 个资源为 r~1~, r~2~, ……, r~m~，规定进程不得在占有资源 r~i~（1 <= i <= m）后再申请 r~j~（j < i），就可以防止死锁。

银行家算法

该算法需要检查申请者对资源的最大需求量，如果系统现存的各类资源可以满足申请者的请求，就满足申请者的请求。

这样申请者就可很快完成其计算，然后释放它占用的资源，从而保证了系统中的所有进程都能完成，所以可避免死锁的发生。

死锁排除

  • 撤销陷于死锁的全部进程；
  • 逐个撤销，直到死锁解除；
  • 从陷于死锁的进程中逐个强迫其放弃占用的资源，直到死锁解除。

Redis

Redis 为什么快？

  • Redis 基于纯内存操作，所有的数据都在内存中，所有的运算都是内存级别的运算。
  • Redis 基于 Reactor 模式设计开发了一套高效的事件处理模型，主要是单线程事件循环和 IO 多路复用。
  • Redis 内置了多种优化过后的数据结构实现，性能非常高。

Redis 缓存

缓存数据处理流程

用户请求 => 缓存中查找（有则返回，否则进行下一步） => 数据库中查找（有则返回，否则返回空数据）

为什么使用 Redis 缓存

高性能：用户第一次请求数据，会从数据库中查询，是从硬盘中读取，较慢。如果用户访问的数据属于高频数据并且不经常改变的话，就可以将数据放在缓存中，提高效率。

高并发：MySQL之类的数据库的 QPS (Query Per Second, 服务器每秒可以执行的查询次数) 大概在 1w 左右（4核8G），使用 Redis 缓存很容易达到 10w+，甚至最高能达到 30w+，使用 Redis 集群会更高。

Redis 线程模型

Redis 读写命令是单线程。4.0 版本后引入多线程执行一些大键值对的异步删除操作，6.0 版本后引入多线程处理网络请求，提高网络 IO 读写性能。

Redis 基于 Reactor 模式开发了自己的网络事件处理器：这个处理器被称为文件事件处理器（file event handler）。

  • 文件事件处理器使用 I/O 多路复用（multiplexing）程序来同时监听多个套接字，并根据套接字目前执行的任务来为套接字关联不同的事件处理器。
  • 当被监听的套接字准备好执行连接应答（accept）、读取（read）、写入（write）、关 闭（close）等操作时，与操作相对应的文件事件就会产生，这时文件事件处理器就会调用套接字之前关联好的事件处理器来处理这些事件。

虽然文件事件处理器以单线程方式运行，但通过使用 I/O 多路复用程序来监听多个套接字，文件事件处理器既实现了高性能的网络通信模型，又可以很好地与 Redis 服务器中其他同样以单线程方式运行的模块进行对接，这保持了 Redis 内部单线程设计的简单性。

Redis 怎么监听大量的客户端连接

Redis 通过 IO 多路复用程序 来监听来自客户端的大量连接（或者说是监听多个 socket），它会将感兴趣的事件及类型（读、写）注册到内核中并监听每个事件是否发生。

这样的好处非常明显： I/O 多路复用技术的使用让 Redis 不需要额外创建多余的线程来监听客户端的大量连接，降低了资源的消耗（和 NIO 中的 Selector 组件很像）。

文件事件处理器（file event handler）主要是包含 4 个部分：

  • 多个 socket（客户端连接）
  • IO 多路复用程序（支持多个客户端连接的关键）
  • 文件事件分派器（将 socket 关联到相应的事件处理器）
  • 事件处理器（连接应答处理器、命令请求处理器、命令回复处理器）
